Instagram Network Analysis

Overview:

This project explores the structure of an individual's Instagram social graph by identifying and analyzing mutual connections among followed users. I primarily looked at the data for my own Instagram network but also examined my sister's and roommate's networks for a comparison point. Future work would include comparing more diverse networks. The pipeline integrates web scraping and network science to uncover community structures, central users, and hidden patterns in online social behavior, as well as graph neural networks to predict potential future connections. The main goals were to make interesting visualizations of social networks, analyze local and global network statistics, and explore how a network can used as a basis for a graph neural network to predict future connections.

Code for this project is available on my GitHub: instagram_network_analysis. The main analysis is in the Jupyter notebook graph_vis3.ipynb, which includes detailed visualizations and network statistics. This idea came from a wolfram alpha tool that was available a decade ago that drew network graphs of facebook friends. I then found and initially cloned Max Impiessen's blog post on Medium. But in the end all of my code was written from scratch as his required significant modifications to work with the current Instagram API and Playwright.

Tech Stack:

  • Python — core programming language
  • Playwright — automated browser control for web scraping
  • NetworkX — graph construction, centrality metrics, and global network analysis
  • python-louvain — community detection via Louvain algorithm
  • PyTorch Geometric — graph neural networks and link prediction
  • Pyvis — interactive HTML visualizations of graph data
  • Pandas — tabular data manipulation
  • NumPy — numerical computations and array processing
  • Scikit-learn (optional/implicit) — for evaluation metrics and normalization
  • Jupyter — interactive notebook environment for analysis and visualization
  • Dash (initially considered) — for interactive web dashboards, but replaced by Pyvis for simplicity
  • GitHub — version control and code sharing
  • HTML/CSS — for embedding visualizations in web pages

Step 0: Data Collection

The first challenge in this project was gathering data from Instagram—a platform known for its aggressive anti-scraping measures. Initial attempts using traditional tools like Selenium proved unreliable due to frequent timeouts, dynamic content loading, and detection. Similarly, Instaloader, while powerful, was quickly throttled and lacked the flexibility needed for collecting mutual follow data at scale.

The solution was to use Playwright, a modern browser automation tool that mimics human behavior more effectively. Even with Playwright, rate-limiting and throttling became a serious issue when collecting data from more than a few dozen users. Strategies such as introducing randomized delays, error handling, and saving intermediate progress were critical to make the process sustainable.

The core data collected was each user's "following" list—who they follow on Instagram. By crawling this data across a target list of users, I constructed a directed graph where an edge from user A to user B represents A following B. Mutual connections were identified as bidirectional edges. The resulting graph structure was saved as a JSON adjacency list, serving as the foundation for all further network analysis and visualization.

Scope was inheritantly limited by Instagram's API restrictions and the need to avoid detection. I would've loved to have scrapped ALL of the followers from each user, but this was not feasible. Instead, I focused on a smaller set of users, primarily my own network to ensure the data was manageable and meaningful. This also allowed for a more focused analysis of mutual connections and community structures within a smaller social circle.

Step 1: Graph Construction

After collecting the raw data, the next step was building a visual and analyzable representation of the Instagram network. Using NetworkX, I constructed a directed graph where nodes represent users and edges represent follow relationships. To position nodes meaningfully in 2D space, I used graph viz's scalable force-directed placement, which is well-suited for visualizing large, organic graphs with natural clustering.

To uncover the underlying social structure, I applied the Louvain community detection algorithm. This grouped users into communities based on follow patterns, allowing for deeper analysis of subgroups and social clusters within the network.

For visualization, I initially experimented with Dash and Dash Cytoscape to create an interactive dashboard that allowed for filtering, highlighting, and analysis in the browser. However, integrating and hosting a Dash app on a static website introduced significant challenges. As a result, I returned to Pyvis, which offered a clean and interactive HTML-based visualization that was easier to embed and share without requiring a backend server.

Step 2: Local Network Statistics

With the graph constructed, I calculated a variety of local network statistics to understand individual user roles and relationships within the network. This was particularly fun as it involved looking at which of my friends played what roles in my greater social network. Key metrics included:

  • Degree Centrality: Measures how many connections a node has relative to all other nodes.
  • In-Degree Centrality: Counts how many edges point toward the node. That is how many people in the graph follow them.
  • Out-Degree Centrality: Counts how many edges originate from the node. That is how many people in the graph they follow.
  • Closeness Centrality: Is the reciprocal of the sum of the shortest path distances from a node to all other nodes.
  • Betweenness Centrality: Measures how often a node appears on the shortest paths between other nodes. It highlights nodes that act as bridges or intermediaries within the network, focusing on the number of shortest paths passing through the node.
  • PageRank: Determines the importance of nodes based on incoming links, similar to Google's PageRank algorithm. It assigns higher scores to nodes that are linked by other high-scoring nodes.
  • Load Centrality: Similar to betweenness centrality, but instead of counting the number of shortest paths, it considers the load or weight of those paths passing through a node. The load is determined by the relative importance or frequency of the paths, emphasizing the flow of information or traffic within the network.
  • HITS Hub Scores: Measure how well a node serves as a hub by linking to authoritative nodes. This is determined by iteratively assigning higher scores to nodes that point to nodes with high authority scores.
  • HITS Authority Scores: Measure how authoritative a node is based on links from hub nodes. This is determined by iteratively assigning higher scores to nodes that are linked by nodes with high hub scores.
  • Clustering Coefficient: Measures the tendency of a node's neighbors to also be connected.
  • Triangles Count: Counts the number of triangles a node is part of.
  • Eigenvector Centrality: Measures influence by considering the importance of a node’s neighbors.
  • Square Clustering Coefficient: Captures how likely neighbors of a node form squares (4-cycles).

Step 3: Global Network Statistics

After analyzing individual users, I turned to the overall structure of the Instagram network. Using NetworkX, I calculated a variety of global statistics that provide insights into the network's connectivity, clustering, and overall structure. Key metrics included:

  • is_directed: Indicates whether the graph is directed (edges have direction).
  • number_of_nodes: Total number of nodes in the graph.
  • number_of_edges: Total number of edges in the graph.
  • density: Ratio of actual edges to possible edges.
  • avg_in_degree: Average number of incoming edges per node.
  • avg_out_degree: Average number of outgoing edges per node.
  • avg_total_degree: Average total degree (in + out) per node.
  • max_in_degree: Maximum number of incoming edges for any node.
  • max_out_degree: Maximum number of outgoing edges for any node.
  • average_clustering: Average local clustering coefficient across all nodes.
  • transitivity: Ratio of triangles to connected triplets in the whole graph.
  • average_neighbor_degree: Average degree of a node’s neighbors.
  • reciprocity: Ratio of mutual edges to total edges in a directed graph.
  • degree_assortativity: Correlation between the degrees of connected nodes.
  • global_efficiency: Average inverse shortest path length across all pairs.
  • num_triangles: Total number of triangles in the graph.
  • diameter: Longest shortest path between any two nodes.
  • avg_shortest_path_length: Average length of shortest paths between all pairs.
  • radius: Minimum eccentricity of any node in the graph where eccentricity is the greatest distance from a node to any other node in the graph.
  • avg_eccentricity: Average maximum shortest path length from each node.
  • degree_entropy: Entropy of the degree distribution, showing variability.
  • num_weakly_connected_components: Number of components if edge direction is ignored.
  • num_strongly_connected_components: Number of components where all nodes are mutually reachable.
